# リポジトリのクローン
ここまでの準備を終えたら、いよいよあしたぼホームページの開発環境を構築します。  
あしたぼホームページは**フロントエンド**と**バックエンド**の2つの部分で構成されています。  
フロントエンドはユーザーが直接触れる部分で、バックエンドはサーバー側で動作し、データの処理や保存を担当します。

:::warning 注意
この手順では、WSLがインストールされ、GitHubで`ashitaboliff`のメンバーとして招待されていることが前提となります。
まだの場合は、先に[WSLの環境構築](../setup/wsl)と[GitHubアカウントの準備](../setup/git)を完了させてください。
:::

## 1. リポジトリのクローン
まず、あしたぼホームページのソースコードをGitHubからクローンします。  
以下のコマンドをWSLのターミナルで実行してください。
```bash
cd ~
git clone git@github.com:ashitaboliff/ashitabo_main.git
```
これにより、`ashitabo_main`というディレクトリが作成され、その中にソースコードがダウンロードされます。

:::tip
[Gitのインストール](./git.mdx)で説明したように *「プログラムをいつだれがどのように変更したか」の変更履歴を管理* しているファイルの集まりを**リポジトリ**と呼びます。  
GitHubで管理されているリポジトリをリモートリポジトリ、あなたのパソコンにクローンしたリポジトリをローカルリポジトリと呼びます。
:::

## 2. クローンしたリポジトリをのぞいてみる
ではクローンできたあしたぼホームページのコードをVSCodeでどう開くかについてみていきましょう。

まず、VSCodeを開きましょう。  
VSCodeを開いたら、左側のメニューにある「リモートエクスプローラー」からWSLのUbuntuに接続しましょう。  
そうしてから左上の「ファイル」→「ファイルでワークスペースを開く」を選択すると、Ubuntuのファイルシステムが見えるはずです。

VSCodeをWindowsで使ったことのある人なら少し慣れない画面が出てきたと思いますが、
とりあえず`home`→`ユーザー名`(WSLのユーザー名)→`ashitabo_main`と進んで、`ashitabo_main`を開けば`ashitabo.code-workspace`というVSCodeのワークスペース機能のためのファイルが見えるはずです。これを開きましょう。

あしたぼホームページのコードは、以下のようなディレクトリ構成になっています。
```
├── app
│   ├── frontend
│   └── backend
├── packages
│   └── types
├── secret/
├── .gitignore
├── README.md
├── .sops.yaml
├── ashitabo.code-workspace
└── ...
```
- `app/frontend`：フロントエンドのコードが入っています。ユーザーが直接触れる部分のコードです。
- `app/backend`：バックエンドのコードが入っています。サーバー側で動作し、データの処理や保存を担当するコードです。
- `packages/types`：フロントエンドとバックエンドで共通で使用される型定義が入っています。

基本的に触るのは`app/frontend`と`app/backend`のコードになります！といっても、最初は`app/frontend`のコードを触ることが多いと思います。
