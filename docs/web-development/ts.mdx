# 5. TypeScriptとは

TypeScriptは、JavaScriptに型付けの概念を追加したプログラミング言語です。
みなさんは「型とかそもそもよくわかってないしだるくね？」と思うかもしれませんが、静的型付け、最高です。

## 5.1 TypeScriptの基本構文
TypeScriptの基本的な構文は以下の通りです。
```typescript
// 変数の宣言
let message: string = "Hello, World!";
const pi: number = 3.14;
// 関数の定義
function greet(name: string): string {
  return "Hello, " + name + "!";
}
// 条件分岐
if (message) {
  console.log(greet("Alice"));
} else {
  console.log("No message");
}
```

おー、JavaScriptとほとんど同じですね。
違いは変数や関数の引数、戻り値に型注釈が追加されている点です。例えば、`message`は`string`型、`pi`は`number`型として宣言されています。
:::tip
TypeScriptは型推論(かっこい（笑）)も行うため、必ずしも型注釈を記述する必要はありません。
:::

:::warning
TypeScriptのコードはこのままでは実行できず、JavaScriptにコンパイル(変換)する必要があります。  
その際に型チェックも行われ、型の不一致がある場合はエラーが発生します。
:::

C言語などでは`int`や`float`などの型を明示的に宣言する必要がありますが、TypeScriptでは型注釈を省略しても型推論によって自動的に型が決定されます。
例えば、以下のように書くこともできます。
```typescript
let message = "Hello, World!"; // TypeScriptがstring型と推論
const pi = 3.14; // TypeScriptがnumber型と推論
```

## 5.2 TypeScriptの利点

TypeScriptの利点を初学者に正確に伝えるのはとても難しく、あなた方も理解しがたいものだと思いますが紹介します。雰囲気で理解してください。

まあ何とでも言えますが、僕の思う利点はほぼ一つで、**型が契約になる**ということです。
例えば、以下のような関数を考えます。
```typescript
function add(a: number, b: number): number {
  return a + b;
}
```
この関数は、2つの`number`型の引数を受け取り、`number`型の結果を返すことを契約しています。
TypeScriptの型システムは、この契約が守られているかをコンパイル時にチェックします。
もし、間違って文字列を渡したり、戻り値を文字列にしようとした場合、TypeScriptはエラーを報告します。
```typescript
add("1", "2"); // エラー: 引数はnumber型でなければならない
```
このように、型が契約として機能することで、コードの信頼性が向上し、バグの発生を防ぐことができます。あとはAIを利用する際にも型情報があると補完が効きやすくなったりします。

## 5.3 TypeScriptの学習
TypeScriptはJavaScriptのスーパーセットであるため、JavaScriptの知識があれば比較的容易に学習できます。
ただ先も述べたように、そのままで実行することが出来ません。なのでまあとりあえずはいいです。雰囲気でつかみましょう。

## 5.4 まとめ
- TypeScriptはJavaScriptに型付けの概念を追加したプログラミング言語
- 型注釈を使用して変数や関数の型を指定できる
- TypeScriptの型システムは、コードの信頼性を向上させ、バグの発生を防ぐのに役立つ
